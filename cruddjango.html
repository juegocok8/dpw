📦 Modelo CATEGORIES

logs/models.py

from django.db import models

class CATEGORIES(models.Model):
    id = models.AutoField(primary_key=True)
    name = models.CharField(max_length=100)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        db_table = 'CATEGORIES'

    def __str__(self):
        return self.name

🎯 Vistas CRUD (CBV)

logs/views.py con vistas genéricas:

from django.views.generic import ListView, CreateView, UpdateView, DeleteView
from django.urls import reverse_lazy
from .models import CATEGORIES

class CategoryList(ListView):
    model = CATEGORIES
    template_name = 'logs/categories_list.html'
    context_object_name = 'categories'

class CategoryCreate(CreateView):
    model = CATEGORIES
    fields = ['name']
    template_name = 'logs/categories_form.html'
    success_url = reverse_lazy('categories_list')

class CategoryUpdate(UpdateView):
    model = CATEGORIES
    fields = ['name']
    template_name = 'logs/categories_form.html'
    success_url = reverse_lazy('categories_list')

class CategoryDelete(DeleteView):
    model = CATEGORIES
    template_name = 'logs/categories_delete.html'
    success_url = reverse_lazy('categories_list')

Ventaja: código mínimo, máximo control.

🔗 Configuración de URLs

logs/urls.py → patrones RESTful:

from django.urls import path
from .views import *

urlpatterns = [
    path('categorias/', CategoryList.as_view(), name='categories_list'),
    path('categorias/nueva/', CategoryCreate.as_view(), name='categories_create'),
    path('categorias/<int:pk>/editar/', CategoryUpdate.as_view(), name='categories_update'),
    path('categorias/<int:pk>/eliminar/', CategoryDelete.as_view(), name='categories_delete'),
]
GET /categorias/ GET/POST /categorias/nueva/ GET/POST /categorias/<id>/editar/ GET/POST /categorias/<id>/eliminar/

Notas importantes:

📊 Ventajas del enfoque Django Class-Based Views

Código mínimo
Cada operación CRUD en ~5 líneas. Comparado con 20+ líneas en vistas basadas en funciones.
Reutilizable
Las CBV heredan comportamientos, se pueden extender con mixins fácilmente.
Rapidez de desarrollo
En minutos tienes un CRUD funcional validado, ideal para prototipos y proyectos reales.

Resultado final: Una aplicación de gestión de categorías completamente funcional, con operaciones crear, leer, actualizar y eliminar, lista para integrar estilos y lógica adicional. Es equivalente a un CRUD en otros frameworks (Laravel, Rails, etc.), pero aprovechando la automatización y seguridad de Django.

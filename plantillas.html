Herencia de Plantillas en Django (DRY)

Don't Repeat Yourself · Reutilización avanzada
Base inteligente → Plantillas hijas limpias y modulares. Ahorra cientos de líneas de código repetitivo.

🧬 ¿Qué es la herencia?

La herencia de plantillas permite reutilizar código HTML en varias páginas, evitando repetir estructuras como encabezados, menús o pies de página.

Plantilla base (padre) → estructura común
Plantillas hijas → extienden y personalizan bloques

DRY Principle Escribe una vez, hereda en todas partes.

📁 Estructura recomendada

Dentro de tu aplicación logs/:

📁logs/
└── 📁templates/
└── 📁logs/
├── base.html
└── inicio.html

Buenas prácticas: anidar en subcarpeta con nombre de la app evita conflictos entre templates.

🏛️ Plantilla base (base.html)

Define la estructura global, headers, footer y bloques editables.

<!DOCTYPE html> <html lang="en"> <head>   <meta charset="UTF-8">   <title>Document</title> </head> <body>   <header>     <h1>Bienvenido a mi pagina web<br> !!!!Texto de la plantilla Base¡¡¡¡¡</h1>   </header>   <main>     {% block contenido %}     {% endblock %}   </main>   <footer>     <p>{{ autor }}</p>   </footer> </body> </html>

block contenido → punto de inyección para páginas hijas.

🌱 Plantilla hija (inicio.html)

Extiende la base y rellena solo el contenido variable.

{% extends 'logs/base.html' %}

{% block contenido %}
<p>
<h1>Contenido dinámico plantilla dependiente</h1>
</p>
{% endblock %}

La ruta en {% extends %} respeta la estructura de carpetas → logs/base.html.

⚙️ Uso desde las vistas y configuración de URLs

📄 logs/views.py → ambas rutas apuntan a la misma plantilla hija

from django.shortcuts import render

def inicio(request):
    return render(request, 'logs/inicio.html')

def contenido(request):
    return render(request, 'logs/inicio.html')

🌐 miweb/urls.py (patrones de URL incluidos)

urlpatterns = [
    path('panel/', admin.site.urls),
    path('', include('logs.urls')),
    path('contenido/', include('logs.urls')),
]

Nota: ambas vistas renderizan inicio.html que hereda de la base, demostrando flexibilidad.

📌 Reglas importantes

  • La ruta en {% extends %} debe coincidir con la estructura de carpetas: 'logs/base.html'.
  • Siempre usar {% block nombre %} para contenido dinámico que puede sobrescribirse.
  • No repetir HTML en cada archivo → DRY esencial.
  • La plantilla hija puede definir múltiples bloques, heredando el resto de la base.

🧩 Alternativa: carpeta global templates

También puedes crear templates a nivel de proyecto:

📁miweb/
└── 📁templates/
└── base.html

Configuración en settings.py:

TEMPLATES = [
    {
        'DIRS': [BASE_DIR / 'templates'],
    },
]

Luego en cualquier template hija:

{% extends 'base.html' %}

✅ Ideal para layouts que comparten múltiples aplicaciones.

📝 Resumen: Herencia en pocas líneas

base.html → plantilla principal con bloques
inicio.html → extiende la base
{% extends %} → hereda la estructura
{% block %} → define contenido dinámico
Ventaja clave: La herencia de plantillas mantiene el código limpio, reutilizable y organizado. Cambia el header una vez en base.html y se refleja en todas las páginas hijas.

📈 Antes vs Después (DRY en acción)

Sin herencia
<!-- MISMO HEADER en 10 páginas -->
<header><h1>Menú</h1></header>
<!-- Contenido diferente -->
... Código duplicado

❌ 1000+ líneas duplicadas

Con herencia (DRY)
{% extends 'base.html' %}
{% block content %}Nuevo contenido{% endblock %}

✅ Código mínimo, máxima reutilización

Consejo profesional: Organiza bloques semánticos como {% block title %}, {% block css %}, {% block js %} para un control granular. La herencia es la base de proyectos Django escalables.

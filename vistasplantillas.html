Vistas y Plantillas en Django

Guía esencial · Estructura moderna
Separación clara entre lógica (Python) y presentación (HTML) → Código mantenible y profesional.

📁 Estructura del Proyecto

Tu proyecto está dividido en dos partes clave:

miweb/ → Proyecto principal (configuración)
logs/ → Aplicación (lógica de negocio)

Convención Django Cada app tiene su propia lógica y plantillas.

📁 miweb/
├── settings.py
├── urls.py
└── wsgi.py
📁 logs/
├── 📁 migrations/
├── views.py
├── models.py
└── 📁 templates/

👁️ Vistas (views.py)

Las vistas contienen la lógica que procesa peticiones y retorna respuestas. Ubicación:

logs/views.py
from django.shortcuts import render

def inicio(request):
    return render(request, 'inicio.html')

Tip: inicio.html esta en 📁 logs/templates/

🔗 URLs: Enrutamiento

3.1 URLs de la App → logs/urls.py

from django.urls import path
from . import views

urlpatterns = [
    path('inicio/', views.inicio, name='inicio'),
]

3.2 URLs del Proyecto → miweb/urls.py

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('logs.urls')),
]

📄 Plantillas (Templates)

Ubicación recomendada:

logs/templates/inicio.html
<!DOCTYPE html>
<html>
<head><title>Inicio</title></head>
<body>
<h1>Hola mundo desde Django</h1>
</body>
</html>

Django busca automáticamente dentro de templates/ de cada app instalada.

📤 Enviar datos a la vista

Vista modificada con contexto:

def inicio(request):
    datos = { 'titulo': 'Bienvenidos a mi pagina', 'contenido': 'Esta es una pagina de prueba', 'autor': 'Ricardo Quintanilla', }
    return render(request, 'inicio.html', datos)

En la plantilla, accedes con la sintaxis de llaves dobles:

<body> <div> <h1> {{titulo}} </h1> </div> <p>{{contenido}}</p> <p>{{autor}}</p> </body>

⚙️ Lógica en Plantillas

{% if %} Condicionales

{% if nombre %}
    <p>✅ Existe nombre: {{ nombre }}</p>
{% endif %}

{% for %} Bucles

<ul>
{% for item in lista %}
    <li>{{ item }}</li>
{% endfor %}
</ul>

También soporta {% else %}, filtros, y más.

🧬 Herencia de Plantillas (DRY)

📄 base.html (estructura madre)

<html>
<head>
  <title>{% block titulo %}{% endblock %}</title>
</head>
<body>
  {% block contenido %}{% endblock %}
</body>
</html>

📄 inicio.html (hija)

{% extends 'base.html' %}

{% block titulo %}Inicio{% endblock %}

{% block contenido %}
<h1>Bienvenido a Django</h1>
{% endblock %}

Evita repetir el layout común

Los bloques pueden ser múltiples ({% block nombre %}) y se sobrescriben en las vistas hijas.

📌 Resumen visual: Estructura y convenciones

Vistas → logs/views.py
URLs app → logs/urls.py
URLs proyecto → miweb/urls.py
Plantillas → logs/templates/
Contexto → Diccionario en render()
Herencia → {% extends %} + bloques
Django separa lógica (Python) y presentación (HTML), facilitando el mantenimiento, la reutilización y la escalabilidad del código.

🎯 Ejemplo integrado: Vista + Bucle + Condicional

Vista (logs/views.py) con lista

def inicio(request):
    articulos = ['📘 Django ORM', '🎨 Tailwind', '🐍 Python Avanzado']
    return render(request, 'inicio.html', {'articulos': articulos, 'usuario': 'Ana'})

Plantilla (inicio.html) usando lógica

{% if usuario %}
    <p>👤 Hola, {{ usuario }}</p>
{% endif %}

<h3>Lista de temas:</h3>
<ul>
{% for item in articulos %}
    <li>{{ item }}</li>
{% endfor %}
</ul>

El motor de templates de Django es seguro y evita inyección de código.

📥 Requisito previo: Descargar Python desde python.org/downloads/ e instalarlo (recordar marcar "Add to PATH").

🚀 Configuración inicial del proyecto

1 & 2. Abrir terminal (PowerShell o CMD) e ir al disco C:\ y crear carpeta 6060: 
cd C:\
mkdir 6060
3 & 4. Acceder a la carpeta: 
cd 6060
5. Crear entorno virtual: 
python -m venv venv
6. Activar el entorno virtual (Windows PowerShell/CMD): 
.\venv\Scripts\activate

⚙️ Configuración de base de datos y adaptador

14. Editar archivo miweb/__init__.py y dejarlo así (inicializar PyMySQL): 
import pymysql
pymysql.version_info = (2, 2, 8, "final", 0)
pymysql.install_as_MySQLdb()
📸 media/image8.png — Contenido final de __init__.py
💡 Importante: Este paso permite que Django se comunique con MySQL usando PyMySQL.
15. Configurar la conexión a MySQL en miweb/settings.py. Reemplazar la configuración DATABASES: 
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'nombre_de_tu_bd',      # Cambiar por tu DB
        'USER': 'root',                 # Usuario MySQL
        'PASSWORD': 'tu_contraseña',    # Contraseña
        'HOST': 'localhost',
        'PORT': '3306',
    }
}

18. Definir los modelos en logs/models.py. Copiar exactamente: 
from django.db import models

class USERS(models.Model):
    id = models.AutoField(primary_key=True)
    name = models.CharField(max_length=100)
    apellido = models.CharField(max_length=100)
    email = models.EmailField(unique=True, max_length=150)
    clave = models.CharField(max_length=255)
    rol = models.CharField(max_length=50)

    class Meta:
        db_table = 'USERS'

class CATEGORIES(models.Model):
    id = models.AutoField(primary_key=True)
    name = models.CharField(max_length=100)
    created_at = models.DateTimeField()
    updated_at = models.DateTimeField()

    class Meta:
        db_table = 'CATEGORIES'

class TASK_LOGS(models.Model):
    id = models.AutoField(primary_key=True)
    user = models.ForeignKey(USERS, on_delete=models.CASCADE, db_column='user_id')
    title = models.CharField(max_length=150)
    description = models.CharField(max_length=255, blank=True, null=True)
    log_date = models.DateField()
    start_time = models.TimeField()
    end_time = models.TimeField()
    duration_minutes = models.IntegerField()
    category = models.ForeignKey(CATEGORIES, on_delete=models.CASCADE, db_column='category_id')
    created_at = models.DateTimeField()
    updated_at = models.DateTimeField()

    class Meta:
        db_table = 'TASK_LOGS'
19. Registrar modelos en el admin: editar logs/admin.py: 
from django.contrib import admin
from .models import USERS, CATEGORIES, TASK_LOGS

admin.site.register(USERS)
admin.site.register(CATEGORIES)
admin.site.register(TASK_LOGS)

🐞 Posibles errores comunes

  • ❌ ModuleNotFoundError: No module named 'pymysql' → activa el entorno virtual y ejecuta pip install pymysql otra vez.
  • ❌ django.core.exceptions.ImproperlyConfigured: 'mysqlclient' → asegúrate de incluir import pymysql; pymysql.install_as_MySQLdb() en miweb/__init__.py.
  • ❌ Error de conexión a MySQL (2003, 1045) → revisa usuario, contraseña, y que MySQL esté corriendo (XAMPP, WAMP o servicio independiente).
  • ❌ Tablas ya existentes → usar --fake-initial o python manage.py migrate --fake si se requiere preservar datos.
